# stat-model-paper 使用说明

> 全国大学生统计建模大赛论文全自动生成 | 14步流水线 | 5种竞赛模板 | 12项终检

---

## 快速开始

复制以下提示词，粘贴到 Claude Code 对话框：

```
/stat-model-paper

【强制规则】
- 严格按 skill 的 14 步管线执行，一步一确认，每步必须输出可验证的中间产物
- 禁止手工等效替代：不允许"我手动搜了文献所以跳过 literature_search.py"
- 每完成一步，报告：哪个脚本跑了、输出了什么文件、下一步是什么
- 管线中标注为 🚫 的禁止项（如脚本硬编码论文、全英文文献）不得违反

【信息】
- 学校：XXXX大学
- 队员（队长在前）：张三、李四、王五
- 指导老师：赵老师
- 选题：学区溢价与住房特征价格的实证分析——基于Hedonic模型的微观证据
- 数据：C:/data/housing.csv
- 因变量：price
- 核心自变量：area, school
- 控制变量：bedroom, age

【交付标准】
- 每步结束后暂停，等我确认再进入下一步
- 最终：Word + 证据校验JSON + 自评报告PDF，全部 >= 85 分
```

---

## 提示词字段说明

| 字段 | 必填 | 说明 | 示例 |
|------|------|------|------|
| 学校 | ✅ | 参赛学校全称 | 北京大学 |
| 队员 | ✅ | 3人，队长在前 | 张三、李四、王五 |
| 指导老师 | ✅ | 1-2人 | 赵老师 |
| 选题 | 否 | 留空随机生成 | 学区溢价的实证分析 |
| 数据 | ✅ | CSV/Excel路径，或写"模拟" | C:/data/housing.csv |
| 因变量 | ✅ | 数据中的目标列名 | price |
| 核心自变量 | ✅ | 最关心的1-2个变量 | area, school |
| 控制变量 | 否 | 其他自变量 | bedroom, age |

## 支持的竞赛类型

| 竞赛 | 说明 |
|------|------|
| 全国大学生统计建模大赛 | 默认，11章 + 双语摘要 + 内生性 + 稳健性 + 政策建议 |
| 市场调查与分析大赛 | 抽样设计 + 问卷 + 信效度检验 |
| 计量经济学论文大赛 | 研究假说 + 识别策略 + 一阶段F值 |
| 能源经济学术论文大赛 | 能源/碳排放数据 + 双碳政策 |
| 挑战杯-社会科学类 | 学术性 + 可读性 + 社会价值 |

如需使用非默认竞赛，在提示词中加上 `竞赛类型：市场调查与分析大赛`。

## 管线步骤速查

```
Step 0    赛题解析        → 识别变量和问题类型
Step 1    智能模型池      → 12模型对比 + VIF + White检验
Step 1.5  模型反省        → R²<0.5强制换方法
Step 2    模板加载        → 5种统计竞赛模板
Step 2.5  文献检索 🔴     → 中英混合，中文≥6篇
Step 3    逐节生成        → 11章，每节字数自检
Step 4    组装初排        → 代码附录≥30行（强制）
Step 4.5  证据校验 🔴     → 论文数值 vs JSON
Step 5    降重处理        → 修改率>40%
Step 6    去AI味终扫      → 统计特有AI味检查
Step 7    写入Word        → 保留格式+图表
Step 7.5  排版合规检查    → 页边距/字体/行距
Step 8    断点续传        → 18步自动保存
Step 9    魔鬼代言人审校  → 12项统计建模专项
Step 9.5  自动修复闭环    → max 3轮
Step 9.7  自评打分        → 7维加权，≥85交付
Step 10   交互修改        → 定向改章节
```

## 常见问题

**Q: 论文太短？**
A: 检查是否违反了「🚫禁止脚本硬编码论文」规则。必须 LLM 逐节生成，每节有字数底线。

**Q: 参考文献全是英文？**
A: 已违反「中文≥6篇」规则。提示词中明确要求了中英混合。

**Q: 想改某章？**
A: 直接说「把 4.2 节改一下」，不需要全文重跑（Step 10 交互修改）。

**Q: 能不能跳过证据校验？**
A: 不能。Step 4.5 是硬关卡，不通过 = 后续步骤全部暂停。
